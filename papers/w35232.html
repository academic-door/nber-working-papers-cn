Designing More Informative Tests: Separating Execution from Recognition

设计更具信息量的测试：将执行能力与识别能力分离

Andrew Caplin, Leo Zhu

理论与方法 全球/多国

Abstract

Tests are widely used to measure ability, yet performance on a test often reflects more than the ability to execute assigned tasks. It also reflects the ability to recognize which tasks are worth attempting, how they should be prioritized, and how effort should be allocated under uncertainty. This paper studies how tests can be designed to separate these capabilities. \nWe model a test as a sequential decision problem. Tasks differ in difficulty, their ordering is uncertain, and examinees may acquire costly information about that ordering before choosing how to proceed. The testing environment is the informational structure surrounding the realized test: in particular, the examinee's beliefs about how task difficulty has been arranged. Performance is therefore generated by an optimal recognition–execution policy, not by execution skill alone. \nThe analysis delivers two negative results. First, even in the simplest two-task environment, a single score exhibits dimensional collapse: distinct combinations of execution skill and recognition capability generate identical expected scores. Second, with three tasks, the relationship between capabilities and scores becomes environment-dependent: changing beliefs about task ordering can change which actions are considered and how capabilities translate into performance. \nThese results imply that standard scores are not generally informative enough to separate the capabilities that generate performance. This matters because scores are used to summarize what individuals can do and to guide downstream decisions about placement, training, and instruction. If a test does not separately reveal execution and recognition, it provides limited guidance about which capability is strong, which is weak, and where improvement should be directed. \nWe then show how more informative tests can be designed. Under a simple communicability constraint, two canonical environments—ordered and randomized tests—induce distinct relationships between capabilities and scores. In an ordered test, recognition is suppressed and performance isolates execution. In a randomized test, recognition is activated and performance reflects both execution and recognition. Observing performance across these environments separates capabilities that are confounded in any single score. \nThe paper reframes testing as a problem of informational design: tests should be designed not only to record performance, but to reveal the distinct capabilities that generate it.

中文摘要

测试被广泛用于衡量能力，然而测试中的表现往往反映的不仅仅是执行指定任务的能力。它还反映了识别哪些任务值得尝试、应如何排序以及在不确定性下如何分配努力的能力。本文研究如何设计测试以分离这些能力。我们将测试建模为一个序贯决策问题。任务难度不同，其顺序不确定，考生在决定如何推进前可以获取关于该顺序的昂贵信息。测试环境是围绕已实现测试的信息结构：特别是考生关于任务难度如何安排的信念。因此，表现是由最优的识别-执行策略生成的，而非仅由执行技能决定。分析得出了两个负面结果。首先，即使在最简单的双任务环境中，单一分数也表现出维度坍缩：执行技能和识别能力的不同组合会产生相同的期望分数。其次，在三个任务的情况下，能力与分数之间的关系变得依赖于环境：改变关于任务顺序的信念会改变哪些行动被考虑以及能力如何转化为表现。这些结果表明，标准分数通常不足以提供足够信息来分离产生表现的能力。这一点很重要，因为分数被用来总结个人能力并指导关于分班、培训和教学的后续决策。如果测试不能分别揭示执行能力和识别能力，它就无法提供关于哪种能力强、哪种能力弱以及改进方向在哪里的有限指导。然后，我们展示了如何设计更具信息量的测试。在简单的可沟通性约束下，两种典型环境——有序测试和随机测试——会诱导出能力与分数之间的不同关系。在有序测试中，识别能力被抑制，表现孤立地反映执行能力。在随机测试中，识别能力被激活，表现同时反映执行能力和识别能力。观察这些环境下的表现可以分离出在任何单一分数中都被混淆的能力。本文将测试重新定义为信息设计问题：测试的设计不仅应记录表现，还应揭示产生表现的不同能力。
